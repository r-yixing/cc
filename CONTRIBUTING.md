# Contributing to Decoded

Thank you for your interest in contributing to Decoded! We welcome contributions from the community.

## How to Contribute

### Reporting Bugs

If you find a bug, please open an issue with:
- A clear description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Screenshots (if applicable)
- Browser/device information

### Suggesting Enhancements

We love new ideas! Please open an issue with:
- A clear description of the enhancement
- Why it would be useful
- Any relevant examples or mockups

### Code Contributions

1. **Fork the repository**
2. **Create a branch**: `git checkout -b feature/YourFeature`
3. **Make your changes**
4. **Test thoroughly**: Open `index.html` in multiple browsers
5. **Commit with clear messages**: `git commit -m 'Add feature: description'`
6. **Push to your fork**: `git push origin feature/YourFeature`
7. **Open a Pull Request**

### Code Style Guidelines

- Use semantic HTML5
- Keep CSS organized and commented
- Write clean, readable JavaScript
- Ensure responsive design works on mobile, tablet, and desktop
- Test accessibility features
- Maintain consistent indentation (2 spaces)

### Testing Checklist

Before submitting a PR, please verify:
- [ ] Website displays correctly on Chrome, Firefox, Safari, and Edge
- [ ] All links work correctly
- [ ] Responsive design works on various screen sizes
- [ ] Animations are smooth and not janky
- [ ] No console errors
- [ ] Images load properly
- [ ] Forms (if any) validate correctly

## Questions?

Feel free to open an issue for any questions about contributing!
